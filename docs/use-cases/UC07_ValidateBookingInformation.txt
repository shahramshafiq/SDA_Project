Fully Dressed Use Case - Validate Booking Information

Use Case Name:                    Validate Booking Information
Scope:                            INNsight Hotel Booking System
Level:                            User-Goal
Primary Actor:                    AI Receptionist System
Stakeholders and Interests:       Customer, Admin

Main Success Scenario
Actor Action                                    | System Response
------------------------------------------------|------------------------------------------------
System receives booking details from customer   | System checks that the room exists
                                                | System verifies the room is available
                                                | System confirms the dates are valid
                                                | System confirms the guest name is provided
System reports that all details are valid       | Booking process continues

Extensions
1. Room does not exist -> System informs the customer and asks for a different room
2. Room is already booked -> System suggests available rooms
3. Dates are invalid -> System asks the customer to re-enter dates

Pre-conditions
Booking details have been provided by the customer

Post-conditions
Booking information is validated or errors are reported to the customer
