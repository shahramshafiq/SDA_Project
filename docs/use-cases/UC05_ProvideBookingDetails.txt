Fully Dressed Use Case - Provide Booking Details

Use Case Name:                    Provide Booking Details
Scope:                            INNsight Hotel Booking System
Level:                            User-Goal
Primary Actor:                    Customer
Stakeholders and Interests:       Customer, AI Receptionist System

Main Success Scenario
Actor Action                                    | System Response
------------------------------------------------|------------------------------------------------
Customer enters their name                      | System records the name
Customer enters preferred room type and dates   | System captures the room preference and dates
Customer reviews the provided details           | System asks for confirmation
Customer confirms the details                   | System acknowledges and stores the information

Extensions
1. Customer provides incomplete details -> System asks for the missing information
2. Customer changes a detail -> System updates the captured information

Pre-conditions
Customer is in a booking or cancellation flow

Post-conditions
All necessary booking details have been captured by the system
