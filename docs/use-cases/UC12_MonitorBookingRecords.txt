Fully Dressed Use Case - Monitor Booking Records

Use Case Name:                    Monitor Booking Records
Scope:                            INNsight Hotel Booking System
Level:                            User-Goal
Primary Actor:                    Staff/Admin
Stakeholders and Interests:       Admin, Customer

Main Success Scenario
Actor Action                                    | System Response
------------------------------------------------|------------------------------------------------
Admin or customer requests to view bookings     | System retrieves all relevant booking records
                                                | System displays the booking records
User reviews the records                        | System allows filtering or searching by date or name

Extensions
1. No booking records found -> System displays a message saying no records exist
2. System is unavailable -> System shows an error and asks to try again later

Pre-conditions
The system is running and booking records exist in the system

Post-conditions
Booking records are displayed to the admin or customer
