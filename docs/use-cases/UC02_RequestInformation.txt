Fully Dressed Use Case - Request Information

Use Case Name:                    Request Information
Scope:                            INNsight Hotel Booking System
Level:                            User-Goal
Primary Actor:                    Customer
Stakeholders and Interests:       Customer, AI Receptionist System

Main Success Scenario
Actor Action                                    | System Response
------------------------------------------------|------------------------------------------------
Customer asks what the system can do            | System lists available options
Customer selects a topic to learn more about    | System provides details on that topic
Customer confirms they understand               | System waits for the next request

Extensions
1. Customer asks an unrecognized question -> System suggests available options
2. Customer provides no input -> System re-displays the list of available features

Pre-conditions
A conversation with the system has already been initiated

Post-conditions
Customer is informed about the available features and options
