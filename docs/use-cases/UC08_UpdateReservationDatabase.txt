Fully Dressed Use Case - Update Reservation Database

Use Case Name:                    Update Reservation Database
Scope:                            INNsight Hotel Booking System
Level:                            User-Goal
Primary Actor:                    Staff/Admin
Stakeholders and Interests:       Customer, Admin, AI Receptionist System

Main Success Scenario
Actor Action                                    | System Response
------------------------------------------------|------------------------------------------------
System receives validated booking details       | System saves the reservation record
                                                | System updates room availability
System confirms the record has been saved       | Customer receives a booking confirmation

Extensions
1. Database is unavailable -> System shows an error and retries
2. Duplicate reservation detected -> System alerts staff and cancels the duplicate

Pre-conditions
Booking information has been validated successfully

Post-conditions
The database is updated with the new reservation and room availability is current
