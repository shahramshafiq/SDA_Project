Fully Dressed Use Case - Manage Room Inventory

Use Case Name:                    Manage Room Inventory
Scope:                            INNsight Hotel Booking System
Level:                            User-Goal
Primary Actor:                    Staff/Admin
Stakeholders and Interests:       Admin

Main Success Scenario
Actor Action                                    | System Response
------------------------------------------------|------------------------------------------------
Admin logs in and accesses room management      | System displays current room inventory
Admin adds, updates, or removes a room          | System validates the room details
Admin confirms the changes                      | System updates the room inventory

Extensions
1. Room number already exists -> System alerts admin and asks for a different number
2. Admin tries to remove a booked room -> System prevents removal and shows a warning

Pre-conditions
Admin is logged in and has access to the room management section

Post-conditions
Room inventory is updated with the latest room details
