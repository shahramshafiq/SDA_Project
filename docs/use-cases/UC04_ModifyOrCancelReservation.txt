Fully Dressed Use Case - Modify or Cancel Reservation

Use Case Name:                    Modify or Cancel Reservation
Scope:                            INNsight Hotel Booking System
Level:                            User-Goal
Primary Actor:                    Customer
Stakeholders and Interests:       Customer, Admin

Main Success Scenario
Actor Action                                    | System Response
------------------------------------------------|------------------------------------------------
Customer requests to modify or cancel           | System asks for booking ID/details
reservation                                     |
Customer provides booking ID                    | System retrieves reservation
Customer confirms modification/cancellation     | System updates reservation database
System confirms successful update               | Customer receives confirmation

Extensions
1. Invalid booking ID -> System shows error
2. Cancellation outside allowed period -> System denies request

Pre-conditions
Existing reservation must exist

Post-conditions
Reservation updated or canceled
