Fully Dressed Use Case - Make Room Reservation

Use Case Name:                    Make Room Reservation
Scope:                            INNsight Hotel Booking System
Level:                            User-Goal
Primary Actor:                    Customer
Stakeholders and Interests:       Customer, Admin

Main Success Scenario
Actor Action                                    | System Response
------------------------------------------------|------------------------------------------------
Customer requests to book a room                | System asks for room type, name, and dates
Customer provides room type, name, and dates    | System checks room availability
Customer confirms booking details               | System creates the reservation
System confirms successful booking              | Customer receives a booking confirmation

Extensions
1. Requested room type is unavailable -> System suggests alternative room types
2. Invalid dates provided -> System asks customer to re-enter dates
3. Customer cancels mid-booking -> System discards the reservation and ends the flow

Pre-conditions
At least one room is available in the system

Post-conditions
A reservation is created and the room is marked as booked
